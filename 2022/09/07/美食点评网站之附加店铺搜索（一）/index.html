美食点评网站之附加店铺搜索（一）

一、背景

网站可以根据用户当前定位，查询在一定范围内的店铺并按照评分进行排名

二、优化前技术实现

  1. 将店铺的位置坐标分为两个字段存放到数据库中，一个字段记录经度，另一个字段记录纬度

  2. 遍历所选地区内所有的餐厅获取其位置信息，然后计算是否符合距离要求。

三、优化后技术实现

以上方式虽然可以实现查询附加店铺的功能，但如果所选地区存在店铺数量过多，那么查询效率就骤降，因此我选择利用R树空间索引来进行优化

（一）什么是R树空间索引

R树就是一棵用来存储高维数据的平衡树，它将B树的思想扩展到多维空间，采用了B树分割空间的思想，并在添加、删除操作时采用合并、分解结点的方法，保证树的平衡性。

R树采用了一种称为MBR(Minimal Bounding Rectangle)的方法，在此我把它译作“最小边界矩形”。从叶子结点开始用矩形（rectangle）将空间框起来，结点越往上，框住的空间就越大，以此对空间进行分割

首先我们假设所有数据都是二维空间下的点，图中标志了R8/R9/R10区域中的数据，别把其所代表的不规则图形看成一个数据，而应该把它看作是多个数据组成的一个区域，我们用R8/R9/R10这些最小边界矩形去框住这个不规则区域。因此R树的15个叶子几点也就代表了最基本的15个最小矩形。

之后就是自底向上操作，我们发现R8，R9，R10三个矩形距离最为靠近，因此就可以用一个更大的矩形R3恰好框住这3个矩形。同样道理，R11，R12被R4恰好框住，R13，R14被R恰好框住……所有最基本的最小边界矩形被框入更大的矩形中之后，再次迭代，用更大的框去框住这些矩形。

如果用地图的例子来解释，那么餐厅代表的就是最基本的最小矩形，之后把相邻的餐厅划分到同一块区域，划分好所有餐厅之后，再把邻近的区域划分到更大的区域，划分完毕后再次进行更高层次的划分，直到划分为几个最大区域，这时候进行查找就方便许多了。

（二）R树结点的结构

  1. 最小外接矩形 Minimum Bounding Rectangle（MBR）
    由于二维及以上的空间数据大多都是不规则的，如果直接对其进行数据建模和数据运算则会比较复杂，所以引入MBR的概念：MBR即能够完全包裹住空间数据对象的最小矩形

  2. R Tree Node
    R树节点，节点中存储得有多个R Tree Entry，R树节点的MBR是包裹该节点里面所有Entry MBR的MBR

  3. R Tree Entry（R树节点中存储的数据对象）

    • 非叶节点中的R Tree Entry存储的是指向其子节点的指针。

    • 叶节点中的R Tree Entry存储的是指向具体数据的指针

（三）R 树搜索

利用 R 树索引进行搜索需要给定一个搜索矩形（即空间范围查询），搜索算法会返回MBR与搜索矩形有交叠的数据，如下图搜索示例返回被R15包裹的空间数据和被R16包裹的空间数据。

具体搜索步骤如下

  1. 从根节点开始，遍历R树节点并进行检查：

    • 如果当前检查节点是非叶节点，检查节点内的每一个MBR是否与搜索矩形相交，如果是，则继续检查该Entry指向的子节点，否则直接剪枝

    • 如果当前检查节点是叶节点，检查节点内的每一个MBR是否与搜索矩形相交，如果是，则将该Entry指向的数据加入结果集，否则直接跳过该Entry

  2. 遍历结束之后之间返回结果集即可

如上图的搜索示例，由于搜索矩形与根节点中R1的MBR就不相交，所以可以直接对R1及其子树进行剪枝，一下子就排除了R1，R3，R4，R5，R8，R9，R10，R11，R12，R13，R14这些Entry，避免了很多无效的搜索。
然后由于搜索矩形与根节点中R2的MBR相交，则继续对R2指向的子节点进行检查，R2子节点中的R6和R7的MBR都与搜索矩形相交，继续向下搜索，R15和R16与搜索矩形相交且处在叶节点，将R15和R16指向的数据加入结果集；R7指向的子节点中的R17，R18，R19都没有与搜索矩形相交叠，直接跳过，至此搜索完毕，返回结果集，结果集中包含R15和R16指向的数据。

（四）R树插入

R树的插入操作与B树的插入操作类似。当新的数据记录需要被添加入叶子结点时，若叶子结点溢出，就需要对叶子结点进行分裂操作。

（五）R树删除

R树的删除操作与B树的删除操作会有所不同，不过同B树一样，会涉及到压缩等操作。相信读者看完以下的伪代码之后会有所体会。R树的删除同样是比较复杂的，需要用到一些辅助函数来完成整个操作
